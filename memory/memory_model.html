From OS point of view

  1. OS has myriad ways of managing its memory, which is closely related to its hardware implementation, different hardware can be better suited to different memory management algorithms

  2. CPU load instructions from memory according to the value of program counter, which implies programs are loaded into memory before run

Memory content overview

  1. The first a few hundred or so (so-called Low-memory), stores a table of pointers (also called interrupt-vector) to interrupt handler routines

  2. Program Instructions The CPU can only load instructions from memory, so any program to run must be stored there

Normal structure (from inner to outer)

  2. Cache
  3. Main memory
